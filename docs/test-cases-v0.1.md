# Rivalt v0.1 テストケース

## 目的

Rivalt の v0.1 公開前に、最小動作版として必要な機能がそろっているかを確認する。

本ドキュメントは、以下を確認するための受け入れテストケース集とする。

- v0.1 完成指標を満たしているか
- 管理者だけが入力機能を使えるか
- 閲覧者はログインなしで主要画面を見られるか
- 試合結果保存が安全に行われるか
- 集計結果が自然に反映されるか

## v0.1 完成指標

v0.1 の完成は、以下を満たした状態とする。

1. 管理者がプレイヤー登録できる
2. 管理者が試合結果を入力できる
3. 順位に応じてポイントが自動計算される
4. Firestore に試合結果が保存される
5. ランキングが自動更新される
6. 試合一覧が見られる
7. 試合詳細が見られる
8. Discord 共有文をコピーできる
9. ログインなしでも閲覧画面を見られる
10. 金銭関連の機能や項目が存在しない

## v0.1 の権限前提

- 管理者: プレイヤー追加、試合結果入力を行える
- 閲覧者: ホーム、ランキング、試合一覧、試合詳細、プレイヤー一覧を見られる
- プレイヤーごとの個別ログイン機能は v0.1 の完成条件には含めない

## テスト前提

- Firebase Authentication で Google ログインが有効になっている
- Firestore が作成済みである
- `.env` に Firebase 設定が入っている
- `.env` に `VITE_ADMIN_UIDS` が設定されている
- `firestore.rules` の管理者 UID が実運用の UID に置き換え済みである
- `npm run build` が通る状態である

## 最重要確認項目

コミット前または公開前に、最低限以下は通す。

- `TC-A6` 管理者 UID でログインし、入力用 UI が表示される
- `TC-B4` 管理者以外の書き込みが Firestore Rules で拒否される
- `TC-C1` 試合登録で `games` と `results` が一括保存される

## B. 認証・権限制御テスト

### TC-B1 〇

- 観点: 管理者設定なし
- 手順: `.env` に `VITE_ADMIN_UIDS` を入れずにログインする
- 期待結果: ログイン自体はできても、管理者 UI は表示されない

### TC-B2 〇

- 観点: 管理者設定あり
- 手順: `.env` に自分の UID を設定し、Vite 再起動後にログインする
- 期待結果: 管理者 UI が表示される

### TC-B3 〇

- 観点: 非管理者ログイン
- 手順: `VITE_ADMIN_UIDS` に含まれない Google アカウントでログインする
- 期待結果: 閲覧はできるが、追加 UI は表示されない

### TC-B4　〇

- 観点: Firestore Rules
- 手順: 管理者 UID ではないアカウントで書き込みを試す
- 期待結果: Firestore Rules で拒否される

### TC-B5　〇

- 観点: 認可整合
- 手順: `.env` の `VITE_ADMIN_UIDS` と `firestore.rules` の UID を比較する
- 期待結果: 同じ UID が設定されている

### TC-B6 〇

- 観点: 非管理者の表示
- 手順: 非管理者アカウントでログインした状態のヘッダーを見る
- 期待結果: ログイン済みだが管理者ではないことが判別できる
- メモ: 現状は改善余地あり

## 補足

- プレイヤーごとのログインと個別権限制御は、現時点では v0.1 の対象外とする
- 将来的に「プレイヤー自身がログインして自分の機能を使う」仕様を入れる場合は、`Player` と Firebase Authentication の紐付け設計を別タスクで追加する
- 本アプリでは、リアルマネー、賭け金、精算、賞金、換金可能ポイントは扱わない
