## 今後の展望

Rivalt は、まず友達内向けに試合結果を記録・共有できる状態から始め、その後シーズン分けや個人成績などを段階的に足していく。

#### 個人成績

- プレイヤー詳細画面（`/players/:playerId`）
- 直近5試合の成績表示
- 優勝率・入賞率・平均順位・連続入賞記録
- プレイヤーメモ（既存 `memo`）の表示
- 一覧・ランキング・試合詳細からの導線

#### ホーム画面

- **お知らせ**（管理者が `/admin/announcements` で CRUD）
- **アクティビティ**（お知らせとは別枠・自動記録）
  - 新メンバー加入（`member_joined`）
  - 試合結果追加（`game_added`）

#### 試合番号・日時

- **第〇戦（`gameNo`）**: `counters/games` で不変採番（削除しても詰め直さない）
- **開催日時**: `date` + 試合追加時に `time` を自動記録（現在時刻・手入力なし）

#### 運用

- **`VITE_LEAGUE_ID`**: 本番 `main` / テスト `dev` などでデータ分離（`docs/setup.md`）
- Firestore Rules: `counters` / `activities` / `announcements`

#### バグ修正（v0.3 に含む）

- **新規登録後に画面が真っ白・遷移できない**
  - 登録直後の `navigate` と `AuthRedirect` の競合を解消（プロフィール反映後にホームへ）
  - 登録バッチ内のアクティビティ失敗で登録ごと落ちないよう分離
  - スプラッシュ後の URL 正規化で `/register` を潰さない
  - `adminLoading` による全画面 Loading で `navigate` が走らない問題を修正（`AuthRedirect` でも登録済みを `/` へ）

### セキュリティ強化（不特定多数・運用者追加時）

友達内公開から運用者が増え、一般メンバーがプロフィールを編集する前提で、以下を段階的に強化する。

- **入力サニタイズ**: 保存前に `<` `>` 等を除去（v0.2 で基本実装済み）。Firestore Rules の文字数・型検証と併用
- **XSS 対策**: React のテキスト表示のみ（`dangerouslySetInnerHTML` 不使用）。将来リッチ表示を足す場合は DOMPurify 等
- **権限の最小化**: 試合・結果は管理者のみ書き込み。プレイヤーは本人ドキュメントのみ更新
- **CSP（Content Security Policy）**: Firebase Hosting デプロイ時にヘッダ設定を検討
- **監査・運用**: 管理者追加はブートストラップ管理者のみ。BAN による無効化で悪用アカウントを止める
- **レート制限**: 必要に応じ Cloud Functions / App Check で書き込み頻度を制限

### 今後も実装しない機能

以下の機能は、法的リスクや運用トラブルを避けるため、今後も実装しません。

- リアルマネー管理
- 賭け金管理
- 精算機能
- 賞金管理
- 換金可能ポイント
- 金銭の授受を前提とした機能

アプリ内ポイントは、あくまでランキング表示のためのスコアとして扱い、金銭的価値を持たないものとします。
