# Rivalt v0.3 テストケース

## 目的

v0.3 で追加した機能（個人成績・お知らせ・アクティビティ・試合番号・日時・リーグ分離）とバグ修正が設計どおり動作し、**v0.2 の回帰が壊れていない**ことを確認する。

設計・リリース概要: [`design-v0.3.md`](./design-v0.3.md) / [`release-v0.3.md`](./release-v0.3.md)

---

## v0.3 のスコープ（完成指標）

次を満たした状態を v0.3 完了とする。

1. **新規登録**後、ホームへ遷移でき、真っ白画面にならない
2. **プレイヤー詳細**（`/players/:playerId`）で成績サマリ・直近5試合が見える
3. **お知らせ**を管理者が CRUD でき、ホームに表示される（0 件時もセクションが分かる）
4. **アクティビティ**に加入・試合追加が自動記録され、ホームに表示される
5. **試合番号**が削除後も詰め直されず、次の試合はカウンター通りに増える
6. **試合日時**が追加時に自動記録され、一覧・詳細・Discord 文に出る
7. **`VITE_LEAGUE_ID`** で本番とテストのデータが分離できる
8. v0.2 の最重要項目（試合編集削除・権限・BAN・ゲスト閲覧など）が引き続き通る

---

## 権限・データ前提

- **ブートストラップ管理者**: `.env` の `VITE_ADMIN_UIDS`
- **追加管理者**: `leagues/{leagueId}/admins/{uid}`
- **一般ユーザー**: ログイン + `/register` で `players/{uid}` 作成
- **リーグ ID**: テスト時は `.env` の `VITE_LEAGUE_ID`（本番想定は `main`）
- Firestore パス例: `leagues/main/...` または `leagues/dev/...`

---

## テスト前提

- v0.2 の前提をすべて満たす（[`test-cases-v0.2.md`](./test-cases-v0.2.md) の「テスト前提」参照）
- **`firestore.rules`（v0.3 拡張分）がデプロイ済み**  
  `npx firebase deploy --only firestore:rules`
- `.env` に **`VITE_LEAGUE_ID=main`**（本番確認時）。ローカル検証用に `dev` を使う場合は投稿・閲覧で同じ値を使う
- `npm run build` が通る
- 検証後に Vite を再起動したビルド／デプロイを使う（`VITE_*` 変更時）

## RG. 新規登録・遷移（v0.3 バグ修正）

### TC-RG1 

- 観点: 登録完了 → ホーム
- 前提: 未登録の Google アカウント
- 手順: `/register` → Google 認証 → 名前・アイコン（任意）→ **登録してはじめる**
- 期待結果:
  - 真っ白画面にならない
  - `/` のホームが表示される
  - ヒーロー付近に「ようこそ、{名前} さん」
  - ヘッダにプレイヤー名またはアイコン

### TC-RG2 

- 観点: 登録済みの再訪
- 前提: TC-RG1 完了済み
- 手順: `/register` を直接開く
- 期待結果: すぐ `/` にリダイレクトされる（登録フォームに留まらない）

### TC-RG3

- 観点: Firestore
- 前提: TC-RG1 完了直後
- 手順: Console で `leagues/{VITE_LEAGUE_ID}/players/{uid}` を確認
- 期待結果: プロフィール doc がある。可能なら `activities` に `member_joined` も 1 件（無くても登録成功は OK）

### TC-RG4

- 観点: アクティビティ失敗時の登録
- 前提: （任意）Rules で `activities` の create を一時的に外して検証する場合のみ
- 期待結果: 登録自体は成功しホームへ行ける（Console に activity エラーのみ）

## GN. 試合番号（第〇戦）

### TC-GN1 〇

- 観点: 採番
- 前提: 管理者、既存試合の最大が第 M 戦
- 手順: 試合を 1 件追加
- 期待結果: 第 **M+1** 戦として表示（件数+1 ではない）

### TC-GN2 〇

- 観点: 削除後の欠番
- 手順: 途中の試合（例: 第 3 戦）を削除 → さらに試合を追加
- 期待結果:
  - 残った試合の番号は変わらない（3 は欠番のまま）
  - 新規はこれまでの最大 +1（例: 5 戦まであれば次は第 6 戦）

### TC-GN3 〇

- 手順: 試合詳細で共有文を確認

## LG. リーグ ID 分離（`VITE_LEAGUE_ID`）

### TC-LG1 〇

- 観点: dev でテスト
- 手順:
  1. `.env` を `VITE_LEAGUE_ID=dev` にし Vite 再起動

- 手順: `dev` でお知らせ投稿 → `main` に切替

## 既知の注意

- **お知らせは自動では出ない**: 管理者が `/admin/announcements` で投稿するまで 0 件
- **アクティビティのバックフィルなし**: v0.3 デプロイ以前の加入・試合はフィードに出ない
- **`VITE_LEAGUE_ID`**: ビルド時に埋め込まれる。本番デプロイは `main` を確認
- **Rules 未デプロイ**: `announcements` / `activities` / `counters` で permission-denied。お知らせ取得エラー表示や登録・採番失敗の原因になる
- **旧試合の `time`**: 無い場合は日付のみ表示
