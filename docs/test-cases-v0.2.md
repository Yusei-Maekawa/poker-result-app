# Poker League Board v0.2 テストケース

## 目的

v0.2 として追加・整理した機能が、設計どおり動作し、v0.1 の回帰も壊していないことを確認する。

## v0.2 のスコープ（完成指標）

次を満たした状態を v0.2 完了とする。

1. 管理者が試合を**編集・削除**できる
2. 試合・プレイヤー入力に**クライアント側バリデーション**がある（未来日・順位連番など）
3. **スマホ向け**のタップ領域・入力まわりが破綻していない
4. **Discord 共有文**の体裁が整っている（コードブロック・メタ情報など）
5. **ランキング表示**が簡潔で矛盾がない（v0.2 での整理内容）
6. **Google ログイン / 新規登録 / プロフィール**により、**1 アカウント = 1 プレイヤー**（`players/{uid}`）で登録できる
7. 管理者のみが試合・結果を書き込める（v0.1 踏襲）。**管理者はプレイヤー BAN（`isActive` 切替）**ができる

## v0.2 の権限前提

- **ブートストラップ管理者**: `.env` の `VITE_ADMIN_UIDS` に含まれる UID
- **追加管理者**: `leagues/main/admins/{uid}` にドキュメントがあるユーザー（ブートストラップ管理者が GUI から追加）
- **一般ユーザー**: ログイン後 `/register` でプロフィール登録。試合の追加・編集・削除は不可
- **閲覧**: 主要画面は未ログインでも閲覧可能（`AuthRedirect` の公開パスに準拠）

## テスト前提

- Firebase Authentication で Google ログインが有効
- Firestore 作成済み、**本リポジトリの `firestore.rules` に相当するルールがデプロイ済み**
- `.env` に Firebase 設定と `VITE_ADMIN_UIDS`（初期管理者）が入っている
- **デプロイ済み Rules のブートストラップ UID** と **`.env` の `VITE_ADMIN_UIDS`** が一致している
- `npm run build` が通る

## 最重要確認項目（リリース前チェック）

- `TC-G2` 管理者で試合編集が保存できる
- `TC-G3` 管理者で試合削除ができる
- `TC-V1` 未来日・順位重複・連番欠けで保存できない
- `TC-G4` 一般ユーザーは試合編集 URL から保存できない（UI 非表示 + Rules で拒否）
- `TC-BN5` 試合の参加者は、アカウント連携済み（`authUid` あり）のプレイヤーのみ選べる

## G. 試合の編集・削除

### TC-G1 〇

- 観点: 導線
- 前提: 管理者でログイン、試合が 1 件以上ある
- 手順: 試合詳細 `/games/:gameId` を開く
- 期待結果: 「編集」「削除」ボタンが表示される

### TC-G2 〇

- 観点: 試合編集
- 前提: 同上
- 手順: 編集へ進み、日付・アプリ名・参加者・順位を変更して保存
- 期待結果: 詳細画面に反映される。Firestore の該当 `games` と `results` が整合している

### TC-G3 〇

- 観点: 試合削除
- 前提: 同上
- 手順: 削除を実行し確認する
- 期待結果: 試合一覧から消える。関連 `results` も削除されている（バッチ仕様）

### TC-G4 〇

- 観点: 非管理者
- 前提: 一般ユーザー（または未ログイン）
- 手順: `/games/:gameId/edit` に直接アクセス、または詳細で編集 UI の有無を確認
- 期待結果: 編集・削除 UI が出ない。可能なら開発者ツール等で書き込みは Rules により拒否される

## V. バリデーション（試合）

### TC-V1 〇

- 観点: 日付
- 手順: 試合作成または編集で、**今日より未来の日付**を選んで保存しようとする
- 期待結果: 「未来の日付は入力できません」等のメッセージで保存できない

### TC-V2 〇

- 手順: 参加者を 1 名のみにして保存
- 期待結果: 2 名未満はエラー

### TC-V3 〇

- 観点: 順位・重複
- 手順: 同一順位を 2 人に付ける
- 期待結果: 重複エラー

### TC-V4 〇

- 観点: 順位・連番
- 手順: 3 名参加で 1,2 を付け 3 を欠く、など連番でない状態にする
- 期待結果: 連番エラー

### TC-V5 〇

- 観点: 文字数（UI）
- 手順: 試合作成・編集でアプリ名・メモを入力する
- 期待結果:
  - ラベル横に **現在文字数 / 上限**（例: `12 / 40`）が表示される
  - 上限付近で色が変わり、上限到達時は枠・「上限に達しています」が分かる
  - `maxLength` によりそれ以上は入力できない（保存前に分かる）

## 回帰（v0.1 相当の抜粋）

次は v0.2 後も毎回軽く通すことを推奨する。

- 未ログインで `/`、`/ranking`, `/games`, `/games/:id`, `/players` が閲覧できる
- 管理者で試合新規作成ができる
- 試合詳細の Discord コピーが動く
- 管理者以外の `games` / `results` 書き込みが Rules で拒否される

## 既知の注意

- **旧データ**: ドキュメント ID が Firebase UID でないプレイヤーは、セルフ登録の対象外。Results にだけ残っていてプレイヤー行を消すと、試合詳細の表示が欠ける場合がある
- **`.env` と Rules**: ブートストラップ管理者 UID は**ビルド時の env** と **デプロイ済み Rules** の両方で一致させること。片方だけだと「アプリでは管理者に見えるが書き込み permission-denied」等が起きる
