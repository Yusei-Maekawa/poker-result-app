# 設計書

## 全体方針

本アプリは、ポーカーゲーム本体ではなく、ポーカーチェイスなどの既存アプリで遊んだ結果を記録する成績管理アプリとして設計する。

## 利用イメージ

1. メンバーが Google アカウントでログインし、プロフィール（プレイヤー）を自身で登録する  
2. 試合終了後に**管理者**が参加者と順位を入力する  
3. ポーカーチェイスなどで友達と対戦する
4. 試合終了後に管理者が順位を入力する
5. ポイントが自動計算される  
6. ランキングが更新される  
7. Discord共有文をコピーして投稿する  

## 権限

### ブートストラップ管理者（初期設定）

- 環境変数 `VITE_ADMIN_UIDS`（カンマ区切り）に含まれる Firebase Authentication の UID  
- Firestore のルール側でも同一 UID を「管理者」として許可する必要がある（運用で二重管理）  
- アプリ上、**他の管理者**を `leagues/{leagueId}/admins/{uid}` に追加できる（GUI）

### 追加管理者（運用管理者）

- `leagues/main/admins/{自分のuid}` ドキュメントが存在するユーザー  
- 試合・結果の読み書き、プレイヤーの BAN（`isActive` の切替）が可能  
- 管理者同士の追加はブートストラップ管理者のみ（現仕様）

### 一般ユーザー（プレイヤー）

- Google ログイン後、`/register` で `leagues/main/players/{auth.uid}` を作成（1 アカウント = 1 プレイヤー）  
- 自身のプロフィール（名前・**プリセットから選ぶアイコン**・メモ）のみ更新可能  
  - アイコンは自由入力ではなく一覧から選択。「自動」で名前の頭文字を表示用に使うことも可  
- 試合の作成・編集・削除は不可  

### 閲覧

- ログインなしでも、ホーム・ランキング・試合一覧・詳細・プレイヤー一覧などを閲覧可能（`/profile` など一部は要ログイン）

## プレイヤーとアカウント

- プレイヤードキュメント ID は **Firebase UID** とする（`players/{uid}`）  
- フィールド `authUid` は本人の UID と一致させる（セルフ登録・更新の整合用）  
- 移行前の旧データは、ID が UID でない・`authUid` 未設定のドキュメントがあり得る。その場合、新規試合の参加者選択には出さないなどアプリ側で区別する  

## プレイヤー BAN

- データの物理削除ではなく、`isActive: false` による**無効化**（管理者のみ）
- 無効プレイヤーはランキング集計・一覧の「アクティブ」表示対象から外す
- **参加者本人には黙って除外しない**: 全画面共通の告知バナーとプロフィール画面で「参加停止・問い合わせ先」を明示する（管理者アカウントではバナーを出さず運用できるようにする）

## ユーザーができること

- ランキング閲覧
- 個人成績閲覧
- 過去試合閲覧

## データ保存方針

データはユーザーごとではなく、リーグごとに保存する。

リーグの中に、プレイヤー、試合、試合結果を持たせる。管理者情報は `leagues/{leagueId}/admins` サブコレクションで管理する。

（シーズン分けは将来バージョンで拡張予定。現状はコード上の定数などで Season 表示のみの箇所がある。）

## セキュリティ（v0.2 時点）

- ユーザー入力（名前・アイコン・メモ・試合メモ等）は **プレーンテキスト** として保存・表示する
- UI は React のデフォルトエスケープのみ使用（HTML 埋め込み表示はしない）
- 保存前に `sanitizeUserText` で `<` `>` や `javascript:` 断片を除去
- 文字数上限は `validationLimits.ts` と Firestore Rules で二重に制限
- 不特定多数が触る前提の追加対策（CSP・App Check・監査ログ等）は `docs/roadmap.md` の「セキュリティ強化」を参照
